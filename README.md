# Smart Travel Companion

Feature-rich travel app built with Flutter + Bloc, backed by Node.js, PostgreSQL, and Redis.

## Documentation Hub

Use this section as the central entry point for all project docs.

- [Quick Start](./QUICKSTART.md) - fastest way to run the project locally
- [Run-First Guide](./RUN_ME_FIRST.md) - pre-flight checks and startup notes
- [Setup Guide](./doc/setup.md) - detailed environment, database, backend, and app setup
- [Features Guide](./doc/features.md) - user-facing features and functional flows
- [Architecture Guide](./doc/architecture.md) - high-level technical architecture
- [Complete Project Guide](./doc/complete-project-guide.md) - full file-by-file explanation, concepts, approach, and design decisions

## Tech Stack

- **Frontend:** Flutter, flutter_bloc, go_router, dio
- **Backend:** Node.js, Express
- **Data:** PostgreSQL, Redis
- **Mapping/Weather:** OpenStreetMap (`flutter_map`), Open-Meteo (via backend proxy)

## Project Structure

- `lib/` - Flutter app code (`core`, `features`, `shared`)
- `server/` - Node.js backend routes, middleware, and database integrations
- `doc/` - architecture, setup, and feature documentation

## Run

Preferred one-command startup:

```bash
./start.sh
```

Manual flow is documented in [Setup Guide](./doc/setup.md).
